<task type="auto">
  <name>Task 2: Create Chrome remote debug detection script</name>
  <files>bin/chrome-detect.sh</files>
  <action>
Create a bash script to detect Chrome remote debugging:

```bash
#!/usr/bin/env bash
# Detect Chrome remote debugging connection
# Returns: JSON with {connected: bool, port: int, ws_url: string}

set -e

PORT="${CHROME_REMOTE_DEBUGGING_PORT:-9222}"
HOST="${CHROME_REMOTE_DEBUGGING_HOST:-localhost}"

# Try to connect to Chrome DevTools
response=$(curl -s -w "%{http_code}" "http://${HOST}:${PORT}/json" 2>/dev/null || echo "000")

if [[ "$response" == "200" ]]; then
    # Extract WebSocket URL from the response
    ws_url=$(echo "$response" | python3 -c "import sys,json; d=json.load(sys.stdin); print(d[0].get('webSocketDebuggerUrl',''))" 2>/dev/null || echo "")
    echo '{"connected": true, "port": '"${PORT}"', "ws_url": "'"${ws_url}"'"}'
    exit 0
else
    echo '{"connected": false, "port": '"${PORT}"', "error": "Chrome not responding"}'
    exit 1
fi
```

This detects if Chrome is running with --remote-debugging-port=9222 (REQ-CH-02).

<task type="auto">

Create a script that launches Chrome with proper --user-data-dir for Chrome 136+:

```bash
#!/usr/bin/env bash
# Launch Chrome with remote debugging for Chrome 136+
# Chrome 136+ requires explicit --user-data-dir

set -e

PORT="${CHROME_REMOTE_DEBUGGING_PORT:-9222}"
CHROME_PATH="${CHROME_PATH:-google-chrome}"
PROFILE_DIR="${CHROME_PROFILE_DIR:-$HOME/.config/google-chrome/remote-debug}"

# Create profile directory if it doesn't exist
mkdir -p "$PROFILE_DIR"

# Launch Chrome with remote debugging
exec "$CHROME_PATH" \
    --remote-debugging-port="$PORT" \
    --user-data-dir="$PROFILE_DIR" \
    --no-first-run \
    --no-default-browser-check \
    "$@"
```

This handles the Chrome 136+ requirement that mandates --user-data-dir (REQ-CH-05).
